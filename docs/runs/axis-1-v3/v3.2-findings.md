## Headline: instruct small_chain flipped — but the nudge never fired

The pre-registered headline cell was **instruct small_chain**. v3.0:
0/3 (the corrective error was delivered, then the model silent-terminated
at the write boundary). The pre-committed rule routed that to v3.2, whose
*one* changed variable was `nudge_budget 0→1` (the S2 mechanism), the
registered lever against silent termination.

**Result: instruct small_chain flipped 0/3 → 3/3. The nudge fired 0 times.**

The flip is entirely attributable to **probe v7's rendering fix**, not to
S2. Under v7 (bare form-B tool messages + LF seeds), instruct now reads
the seed and, on turn 2, writes the 62 bytes with real newlines and calls
mark_complete in the same turn — a clean chain. There is no silent-
terminate turn for the nudge to catch, because there is no failure to
recover from. The mechanism the campaign was built to test was inert on
the very cell it was built to test.

> **The rendering fix, not the nudge, flipped instruct.** v3.2 rode a
> substrate change (v3.0 was probe v3; v3.2 is probe v7) that the
> pre-registration predates. The headline moved for a reason orthogonal
> to the registered hypothesis.

## Did S2 ever earn a pass? No.

The nudge fires only on a **0-tool-call turn with the output still
unwritten**. Across the whole grid, **not one cell's pass is attributable
to it**:

- **instruct / gemma**: 0 nudges. Every pass is a clean v7 chain; every
  failure (instruct loop_depth) is a *non-empty* turn (re-calling
  mark_complete), so the nudge condition is never met.
- **coder**: 6 nudges, **all on loop_depth**. small_chain passed 3/3 with
  zero nudges. Where the nudge *did* fire, it re-activated the model
  (empty turn → nudge → `read_file` → `mark_complete`) — but loop_depth
  still scored **0/3**. The nudge un-stuck the model; the retried write
  was still byte-wrong.
- **nemo**: 0 nudges despite 0/9. nemo never produces an empty turn — it
  re-calls `mark_complete` on every iteration until max-iter. Same
  structural inertness as instruct's loop_depth.
- **qwen3**: 4 nudges, categorical/bistable; no pass traceable to them.

**The nudge is either structurally inert (models loop on `mark_complete`,
never emitting the empty turn S2 keys on) or ineffective (it fires, the
model acts, but the resulting bytes are still wrong).** The S2 condition
is too narrow for the failure population probe v7 actually produces: v7
turned the dominant failure from *silent termination* into *confident
wrong-byte completion*, which S2 does not target.

## Byte-level failure taxonomy (first repeat per block)

Under v7 the copies are no longer rendering artifacts — these are genuine
model byte errors:

| task | expected | instruct | qwen3 | nemo | coder | gemma |
|------|----------|----------|-------|------|-------|-------|
| small_chain (62 B, trailing `\n`) | 62 B ✓ | 62 ✓ | 62 ✓ | **61** (−`\n`) | 62 ✓ | 62 ✓ |
| loop_depth (55 B, a+b) | 55 B ✓ | **54** (−`\n`) | 55 ✓ | **21** (dropped b) | **56** (double `\n`) | 55 ✓ |
| content_robust. (50 B) | 50 B ✓ | 50 ✓ | **49** (−`\n`) | **51** literal `\n` | 50 B ✓ but **no mark_complete** | 50 ✓ |

Failure modes are individually diagnostic:

- **trailing-newline off-by-one** — instruct loop_depth, nemo small_chain,
  qwen3 content_robustness. The single most common error under v7.
- **dropped operand** — nemo loop_depth wrote only seed_a (21 B), never
  concatenated seed_b.
- **spurious separator** — coder inserted a double newline at the a/b join
  (`avocado\n\nblueberry`, 56 B).
- **literal escape** — nemo content_robustness wrote backslash-`n` as two
  characters (51 B) — the only model still emitting escapes under v7.
- **completion-protocol omission** — coder content_robustness wrote the
  target *byte-exact* (50 B, file_content + file_exists pass) but
  max-iter'd without ever calling mark_complete; fails on
  `mark_complete_called` alone. Correct work, unfinished protocol.

## What this settles, and what it opens

- **Settled**: The v3.0→v3.2 "silent termination" story was a probe-v3
  rendering artifact for the copy tasks. Once v7 renders tool results
  plainly, instruct/coder/gemma *write correctly*; the residual failures
  are ordinary byte errors (trailing newline, separators, dropped
  operands), not a harness-recoverable silent-terminate class. S2 as
  specified recovers none of them.
- **Open**: The remaining spread (gemma 9/9 vs the rest) is now a clean
  model-fidelity signal, not a harness confound. The registered next
  lever would target the *actual* residual — trailing-newline and
  separator errors on the concat/redirect tasks — which is a prompt- or
  validation-shaped intervention (e.g. a byte-count/tail-newline
  corrective postcondition), not the empty-turn nudge. That is a new
  pre-registration, not a continuation of the S2 line.

## Erratum (2026-07-03, post-v8 regression forensics)

The claim "S2 earns no pass anywhere" is WRONG. Regression forensics
(docs/runs/axis-1-v8/regression-forensics.md) show coder-14b small_chain
3/3 was nudge-DEPENDENT: both campaigns produce an identical 0-tool-call
stall turn; v3.2's nudge converted it into the write, v8 (nudge off) did
not, and the cell fell 3/3 -> 0/3. The v8 campaign inadvertently ran as
an S2 ablation (two variables changed vs v3.2 — an acknowledged design
error) and exposed the accounting gap.

Revised S2 disposition: the nudge earns passes on EMPTY-TURN STALLS
(coder's failure mode) and is inert on re-signaling and wrong-byte
completion (instruct/nemo). Mechanisms sort by failure mode:
corrective errors -> malformed calls; nudge -> stalls; review ->
falsified (perception floor). Production harness config is corrected to
{tool_errors: corrective, nudge_budget: 1, review_budget: 0}; roles.yaml
implementer harness updated accordingly. coder-14b's casting
ineligibility stands — a profile whose sole pass is mechanism-dependent
is not a stable implementer, but the mechanism itself is rehabilitated
for production use.
